CS 340 Portfolio Reflection

How do you write programs that are maintainable, readable, and adaptable?

I focus on keeping my code organized and easy to understand by using clear variable names, comments, and separating logic into different parts. In this project, the CRUD Python module helped a lot because it kept all database operations in one place. This made the dashboard code cleaner and easier to manage. It also made the program more adaptable since I could reuse the same module in other projects that need database access.


How do you approach a problem as a computer scientist?

I start by breaking the problem down into smaller pieces and focusing on one part at a time. For this project, I first made sure I could connect to the database and retrieve data. Then I worked on the dashboard step by step, adding the table, filters, and charts. This approach was different from some earlier assignments because this project required connecting multiple parts together instead of just writing a single program. In the future, I would continue using this step by step approach and test each part as I build it.


What do computer scientists do, and why does it matter?

Computer scientists solve problems by creating systems that help people work more efficiently. In this project, the dashboard allows a company like Grazioso Salvare to quickly filter and analyze animal data instead of going through it manually. This saves time and helps them make better decisions when selecting animals for rescue training. Work like this matters because it improves how organizations use data to support real world goals.
